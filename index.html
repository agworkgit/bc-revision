Front End Dev - Revision Notes

What does MVP mean?

MVP stands for minimum viable product. This is usually the first iteration of a product without extra features. It's there to prove a solution works.

What does blocker or bottleneck mean?

During development, you may hear the term blocker, which is just another way of saying that you are stuck on a particular problem
that is preventing you from moving forward. Another term you may see is bottleneck, which refers to a point in our development process
that is slowing down the rest of production.

What does URL mean?

URL is a fundamental web development concept. A URL, or Uniform Resource Locator, is the unique address given to a specific resource on the web.
When we deploy a website and make it available to a user, the user accesses the page via a URL such as "google.com".

What does separation of concerns mean?

Separation of concerns is a widely used term in web development and refers to separating code into different files based on use.
It is one of the key design principles that we practice in the boot camp. Separation of concerns is based on the idea that each section
of code should have its own responsibility.

Computational Thinking

What is mean by computation thinking is the idea of decomposition, abstraction (else), pattern recognition, and testing algorithms (debugging).
This helps us to digest easier the larger problem and come up with a solution effectively.

Package Managers

A package contains code written by others that we can add to and use on our computers and in our projects.
A package manager, automates the process of installing, upgrading, configuring, and removing packages.

Terminal Commands

We use terminal commands to quickly navigate between folders, create folders, create files and easily push updates to GitHub.
Here are a few useful commands to get you started:

  • pwd - print working (current) directory
  • ls - list files (in the directory)
  • cd (followed by the name) - changes directory
  • cd .. - moves you back on step
  • mkdir (followed by given name)- make directory (folder)
  • touch (followed by name and file type) - creates a file in the directory selected
  • open - will open a specified file in the folder ie: text-file.txt
  • open . - open current folder in a window
  • code . - open current directory in your default code editor

HTML

Meaning Hyper Text Markup Language is used to add content and structure to our webpage.
It is also the language that gives instructions to the browser with meta tags on what language to use,
how to display the viewport, keywords for SEO, descirption, author and much more.

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.

CSS

Meaning Cascading Style Sheets.
CSS defines the colors, aesthetics, visual layout, and organization.
We use CSS to define style rules for specific elements of an HTML page and format them using a wide range of style options.
There are three ways to style a webpage using CSS:

  • inline CSS, meaning directly in the HTML file
  • internal CSS style sheet, contains CSS rules for the webpage in the head section of the HTML file
  • external CSS style sheet, we link the file using a link element placed in the HTML's head
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

* is used as a wildcard selector, with this selector you can add declarations to all the items on your page.

Rather than repeating a rule which would add more lines of code and repetition, two or more elements
that will share the same styling can be grouped together by using a comma.
This helps us write Do Not Repeat Yourself (DRY) code.

Do Not Repeat Yourself (DRY) is a coding principle that encourages developers to reuse or share pieces of code
in order to reduce the number of lines of code that need to be written or maintained. DRY can greatly reduce the overall size and complexity of a codebase.

Git

An online repository storage server that allows for collaboration, version control, projects and deployment, to showcase your work to potential employers.
Push means upload, pull means download.
GitHub connects to a secure shell or SSH that can store your credentials to quickly send changes to your account via terminal commands.

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it

GitHub's Issues

Performs the function of a task manager, we give it clear user stories and acceptance criteria and work thorough the list until the project is complete and we have an MVP.


GitHub's Pull Request

A pull request, or PR, is used to merge a comparison branch (in our case, the feature branch) into the base branch.

Debugging

Debugging is one of the most important parts of the development process. Every day, developers notice errors and review code that isn't working.
It's important to get into the mindset that errors and mistakes are gifts. They tell us when something is wrong, and if we can stay curious,
we can figure out what is happening and learn something new.

  • Plan ahead on understanding where your code will be used and optimise for every user and device.
  • You can use a browser's console to inspect and make temporary changes that can later be commited to your working files.
  • It's recommended to get feedback from the user that encountered the bug and try to replicate their error on your machine step by step to find the bug.
  • If all the other steps have failed, communicate with the user and get them to share their screen so you can observe what is happening.
  • Learn how to use Visual Studio Code's debugging interface as it will save you a huge amount of time when you're going through the process of finding the bug.
